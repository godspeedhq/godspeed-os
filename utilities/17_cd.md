# Utility: `cd` - change current location

**Status:** **Built + QEMU-verified** (`osdev test files` 11/11) - a shell built-in; the
current location lives in the shell session, which resolves relative / `.` / `..` paths to
an absolute path before calling `fs` (validates the target via STAT). On hierarchical GSFS
(`docs/persistence.md`). Trails `CLAUDE.md`; does not amend it.

---

## 1. What it is

`cd` moves the **current location** - the single pointer that says *where you are*. Because
GSFS is hierarchical, "where you are" is one address: a **drive + directory**. `cd` takes a
full drive-qualified path, so it is the *only* location pointer - there is no separate
"current drive" verb (no `mount`, no `drives use`; see `15_drives.md` §3.1/§3.2 for why
those POSIX reflexes were dropped). `cd` is kept as a verb (universal, not cryptic).

## 2. Usage

```
cd 0.1.0 - change the current location (drive + directory)

usage:
  cd <path>           move to <path>
  cd -                move to the previous directory
  cd version          print the version
  cd help             print this message

<path> = [index:]label/path | /abs | rel | ..   (see docs/drives.md §4.1)
```

Examples:

```
cd 1:data/projects   current = drive 1 (data), dir /projects
cd /etc              current drive, absolute path
cd ../bin            relative, up one
cd archive           the 'archive' drive's root
```

## 3. The current location (session state)

The current location lives in the **shell session** - the shell prefixes it onto relative
paths before sending requests to `fs`, so `fs` always receives a resolved address. The
session also remembers the **previous** location (a one-slot history) so `cd -` can toggle
back. Both are session state: they do **not** persist across boots in the first cut. A
persistent "start here" working default is deferred (`15_drives.md` §5/§3.2) - decided
alongside multi-drive, not invented now.

## 4. Implementation

A **shell built-in**: it resolves the target, `Stat`s it through `fs` (op 12) to confirm it
exists and is a directory, and updates the session pointer. Read-only on disk. `cd -` reuses
the saved previous absolute path and runs the same stat-validated switch (so a since-deleted
previous directory errors loudly rather than silently failing); each successful `cd` saves
the directory it left, so `cd -` twice returns you where you started.
