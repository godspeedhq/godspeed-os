# Utility: `cd` — change current location

**Status:** Design — built in the file-commands step (step 4) on hierarchical GSFS
(`docs/persistence.md`). Read-only (validates the target exists); works on Phase-2 GSFS.
Trails `CLAUDE.md`; does not amend it.

---

## 1. What it is

`cd` moves the **current location** — the single pointer that says *where you are*. Because
GSFS is hierarchical, "where you are" is one address: a **drive + directory**. `cd` takes a
full drive-qualified path, so it is the *only* location pointer — there is no separate
"current drive" verb (no `mount`, no `drives use`; see `15_drives.md` §3.1/§3.2 for why
those POSIX reflexes were dropped). `cd` is kept as a verb (universal, not cryptic).

## 2. Usage

```
cd 0.1.0 — change the current location (drive + directory)

usage:
  cd <path>           move to <path>
  cd version          print the version
  cd help             print this message

<path> = [index:]label/path | /abs | rel | ..   (see docs/drives.md §4.1)
```

Examples:

```
cd 1:data/projects   current = drive 1 (data), dir /projects
cd /etc              current drive, absolute path
cd ../bin            relative, up one
cd archive           the 'archive' drive's root
```

## 3. The current location (session state)

The current location lives in the **shell session** — the shell prefixes it onto relative
paths before sending requests to `fs`, so `fs` always receives a resolved address. It is
session state: it does **not** persist across boots in the first cut. A persistent
"start here" working default is deferred (`15_drives.md` §5/§3.2) — decided alongside
multi-drive, not invented now.

## 4. Implementation

A **shell built-in**: it resolves the target, `Stat`s it through `fs` (op 12) to confirm it
exists and is a directory, and updates the session pointer. Read-only on disk.

## 5. Later (separate doc so it can grow)

- `cd` with no argument → a defined "home" (proposed: the current drive's root — there is
  no POSIX `$HOME` here, so this is a deliberate choice, not an import).
- `cd -` → previous location (a one-slot history).
- Tab-completion of path components (a shell-ergonomics feature, future).

## 6. Conformance

Built spec-first against `0_conventions.md`: implements its own `cd help` / `cd version`.
